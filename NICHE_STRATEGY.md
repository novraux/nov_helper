# Novraux — Niche Strategy
> Phase focus: Find 1–3 winning niches before flooding the store with designs.
> Created: Feb 22, 2026

---

## The Core Problem

Launching a POD store without a niche strategy is how you end up with 50 random designs
and zero sales. The goal of this phase is to **pick focused niches first**, then design into them
with intent — not the other way around.

A good niche gives you:
- A repeatable customer (not a one-time shopper)
- A clear design direction (so you know what to make next)
- SEO leverage (you rank for a cluster of related keywords, not isolated terms)
- Gifting occasions (birthday, Christmas, graduation — huge for Etsy)

---

## What Makes a Niche "Winning"

Score each candidate on these 6 axes (1–5 each, max 30):

| Axis | What to measure |
|------|----------------|
| **Demand** | Are people actively searching? (Google Trends, Etsy volume) |
| **Competition gap** | Is the top of search monopolized by big stores, or is there room? |
| **Design surface** | Can you make 20+ distinct designs without running out of angles? |
| **Buyer intent** | Do they buy for themselves, as gifts, or both? (gifts = higher AOV) |
| **IP safety** | No brand names, mascots, sports teams, or copyrighted phrases |
| **Evergreen** | Is it demand year-round, or only seasonal? (seasonal = hard to build on) |

**Minimum viable score: 22/30** — below that, skip or revisit later.

## Research Workflow — What to Do Before Designing Anything

### Step 1: Validate demand on Etsy (15 min per niche)
1. Search the niche keyword on Etsy (e.g., "veterinary technician gift")
2. Note: total listing count, average price of top 10, whether top sellers have 1000+ sales
3. Look at tags on top listings — what are they using?
4. Check if any shop "owns" the niche (>50 listings, >500 sales in category)

### Step 2: Check Google Trends
1. Compare 3–5 niche variations over 12 months
2. Look for: steady interest (not dying), no extreme seasonality, rising slope preferred
3. Compare related queries — find the long-tail versions with less competition

### Step 3: Check Redbubble + Merch by Amazon
- Redbubble: search niche, sort by "Top Selling" — see what design styles dominate
- Merch by Amazon: BSR (Best Seller Rank) on category pages shows real sales velocity
- These are competitor validation, not copy targets

### Step 4: Reddit/Facebook community check
- Is there a subreddit or Facebook group for this niche?
- Size matters: 10k+ members = real audience
- Check what they post — recurring phrases, inside jokes, pain points = design ideas

### Step 5: Phrase mining
- Use Etsy autocomplete: type "vet tech" and see every autofill suggestion
- Use AnswerThePublic or AlsoAsked for question-based phrases
- These become your design text AND your SEO tags

## Tools — Build Into the Platform (Priority Order)

These are additions to the existing Novraux backend/dashboard:

### Priority 1 — Etsy Autocomplete Scraper
**What**: Hit Etsy's search API with a root keyword, collect all autocomplete suggestions.
No API key needed — uses public endpoint.
**Output**: A ranked list of long-tail keyword variations with estimated competition.
**Why now**: This is the fastest way to find specific design phrases that buyers are already
searching. Takes ~1 day to build.

```
Endpoint: POST /research/etsy-autocomplete
Input: { "keyword": "vet tech" }
Output: ["vet tech gift", "vet tech shirt", "vet tech appreciation", ...]
```

### Priority 2 — Niche Validator
**What**: Takes a niche keyword and returns a validation report:
- Google Trends score (already have pytrends)
- Etsy listing count (proxy for competition)
- Top 10 competitor tags
- AI-generated design angle suggestions (Groq)
- Overall viability score

**Why now**: Replaces manual research with one-click analysis. Core to this phase.

```
Endpoint: POST /research/validate-niche
Input: { "niche": "veterinary technician gifts" }
Output: { score, trends_data, etsy_count, top_tags, design_angles[] }
```

### Priority 3 — Competitor Listing Analyzer
**What**: Given an Etsy listing URL or keyword, scrape the top 10 listings and return:
- Common tags across all top listings
- Average price
- Listing age vs. sales velocity
- Design style descriptions (typography, illustration, etc.)

**Why now**: Shows you what's working in a niche before you design anything.

### Priority 4 — Seasonal Calendar Generator
**What**: For a given niche, generate a list of upcoming events/holidays that are relevant,
with dates and design prompt suggestions.
**Example**: "Vet Tech Appreciation Week = May 12–18. Design angle: 'The real doctors wear scrubs'"

### Priority 5 — Niche Shortlist Dashboard View
**What**: A new tab in the React frontend showing:
- All validated niches with scores
- Status: Research / Designing / Live / Archive
- Number of designs per niche
- Revenue attributed to each niche (once orders sync is built)

## Design Volume Target Per Niche

Before going live with a niche, aim for this minimum:

| Product type | Target designs |
|---|---|
| T-shirts (core) | 8–10 designs |
| Sweatshirts/Hoodies | Same designs, different mockups |
| Mugs | 4–5 designs (not all translate) |
| Stickers | 4–5 designs |

**Why 8–10?** Below this, the Etsy algorithm has nothing to work with. Shops with 1–2
listings in a niche rarely appear in search. Volume signals relevance.

## What Not To Do

- **Do not** list a design in every niche simultaneously — you'll rank for nothing
- **Do not** copy top-selling design text verbatim — Etsy will detect it and it's not differentiated
- **Do not** skip the research step because a niche "feels right" — validate the data first
- **Do not** build 20 designs before you have one sale — validate with 5, then scale
- **Do not** chase trends that are already declining (check the Google Trends slope, not the peak)
